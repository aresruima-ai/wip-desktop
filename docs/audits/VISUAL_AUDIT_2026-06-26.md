## 一、跨页共性问题（优先级最高，全站统一收口）

### P0-1 数据诚实性：加载态 / 无数据 / 0 三态混乱 ⚠️ 全站
最严重共性问题。"--" / 骨架屏白杠 / "0" / "暂无" 在同一屏混用，用户分不清是"系统故障"还是"真无数据"。
- **portal**：右侧小卡 "--" 与左侧 "0" 同屏，口径不一致
- **oee**：满屏 "--" + 底部 "OEE null%"（暴露变量名），像系统故障
- **bad**：FPY 卡产量 0 却标红"严重低于目标"，与左侧"品质良好"矛盾；P90 显示 "--" 却判定"达标"
- **scroll-board**：今日总产出是骨架屏，但下方单位/对比已渲染；FPY/OEE/UPPH 三个核心指标都是骨架屏
- **wip / health / factory-3d**：图表空白无占位引导
→ **全站统一三态规范**：`加载中(骨架屏)` / `无数据(明确"暂无数据"+插画)` / `有值`，禁止 0 与 -- 混用、禁止无数据却下结论。

### P0-2 对比度：次要文字过暗，暗色背景下难读 ⚠️ 全站
几乎所有页面的辅助文字、副标题、表格正文都过暗。
- portal 顶部"数据状态"提示、oee 卡片副标题"目标85%/更新于"、kanban"目标0/低于60%紧急"、admin 表格正文与操作按钮、settings 顶栏菜单与控件、ai-center "FPY 80.07%"等小字、factory-3d 右侧面板
→ 次要文字亮度统一提至 `#B0B0B0+`，操作按钮改实心填充。

### P0-3 色彩语义滥用：0/正常也标红，刺眼色喧宾夺主
- **cockpit**：不良 PPM 数值 0(正常)却用红色边框/元素，误导为报警
- **bad**：产量 0 时 FPY 标红"严重低于目标"
- **wip**：底部警告图标三角/圆点/边框颜色缺乏统一语义
- **fixture-life**：左侧主标题亮绿色大字号刺眼喧宾夺主
- **kanban**：右侧达成率卡片深红背景与暗色主题不协调
→ 红色仅用于真实越界报警；0/正常用中性色；刺眼饱和色降一档。

### P1-1 焦点层次与排版：信息过载、卡片样式不统一
- **portal**：左上焦点卡堆砌产量/PPM/待办/风险/元数据，层次混乱
- **wip**：左侧主卡标题红白混排（红字暗背景对比度极差）
- **kanban**：左侧主卡标题+建议+标签+工单+按钮全堆一起，无分组
- **line-balance**：6 个底部卡片样式/对齐/结构完全不统一（有的圆环、有的"暂无数据"）
→ 焦点卡只保留核心数；周边小卡统一模板（标题/数值/说明布局一致）。

### P1-2 字体割裂：衬线体突兀
- **ai-center**：主标题"品质不良..."用衬线宋体，与全站无衬线冲突
- **settings**：主标题"系统设置"衬线体过大
→ 全站统一无衬线，标题字号收口。

## 二、各页具体问题

### portal（首页）
1. 右侧小卡 "--" 与左侧 "0" 口径不一致 → 统一无数据样式
2. 左上焦点卡信息过载(产量/PPM/待办/风险/元数据) → 拆分，主卡仅核心产量
3. 顶部"数据状态"提示文字对比度过低
4. 左下直通率折线断续、06-21 等日期缺失 → 缺数据处留白/虚线
5. 右下"产线产出对比"完全空白仅"暂无" → 移除或替换

### cockpit（驾驶舱）
1. 左侧主卡面积过大但内容稀疏(仅大标题+圆环)，与右侧密集布局失衡
2. 主标题"全域实时 • OEE -- • PPM 0"字体过大、连字符与圆点混用杂乱
3. 不良 PPM 数值 0 却红色边框，误导报警

### wip（在制品）
1. 标题红白混排，红字暗背景对比度极差
2. 圆环图(工单达成)线条过细无填充，0% 状态几乎不可见
3. 底部指标卡警告图标(三角/圆点)与边框颜色无统一语义

### bad（工程不良）
1. FPY 卡产量 0 却标红"严重低于目标"，与"品质良好"矛盾 → 0 时显示 "--"
2. P90 显示 "--" 却判"达标" → 无数据不可下结论
3. 二维分类筛选不清晰，仅平铺下拉，无法直观交叉分析
4. 误测率 72.9% 未在顶部高亮警告(测试线问题被"真不良"默认筛选掩盖)
5. "无过站数据"与"暂无真实过站数据"文案重复，数值位置不统一

### oee（OEE）
1. 满屏 "--" 占位，像系统故障 → 骨架屏或"暂无数据"空状态
2. 底部 "OEE null%" 暴露变量名 → "数据缺失"或"--"
3. 左侧主卡空状态文字过大缺交互 → 缩小+加"重置筛选"按钮
4. 卡片副标题"目标85%/更新于"对比度过低
5. 底部指标行数值"--"居中、标签左对齐，重心不稳

### kanban（工作看板）

2. 左侧主卡信息过载无分组
3. 右侧达成率卡深红背景与主题不协调，红底文字对比度也差

### uph（UPH 产出）

2. 整体排版拥挤，区块缺呼吸感

3. 右侧面板与顶部文字过暗、字号过小，对比度不足

### ai-center（AI 中心）
1. 主标题衬线宋体与全站无衬线冲突
2. "FPY 80.07%/规则+AI聚合"等辅助文字过浅难读
3. 右侧"未处理"卡深橄榄/褐色背景显脏，与科技感主题不融

### scroll-board（滚动看板）
1. 左侧"今日总产出"是骨架屏但下方单位/对比已渲染 → 加载态不一致
2. 右侧 FPY/OEE/UPPH 三个核心指标都是骨架屏白杠
3. 底部"今日问题清单"序号"1"字号过大颜色突兀喧宾夺主
4. 顶栏主标题与系统状态(班次/同步/时间)挤同一行无分区

### admin（系统管理）
1. 表头左对齐但内容(如 admin)右对齐，动线断裂 → 统一左对齐
2. 表格正文文字过暗对比度极低
3. "编辑/重置密码"按钮边框与文字过深几乎不可见
4. 顶部统计栏"用户2"等字号过小暗淡

### settings（系统设置）
1. 顶栏菜单文字过暗对比度严重不足 → 提亮至 #B0B0B0+
2. 输入框/下拉框背景与卡片背景几乎一致，边界难辨
3. "保存设置"主按钮视觉权重过低(淡蓝边框) → 实心填充
4. 主标题"系统设置"衬线体过大 → 无衬线+缩小

## 三、建议修复顺序
1. **P0-1 数据三态规范**：先立规范(加载/无数据/有值)，全站统一组件化 — 影响可信度最大
2. **P0-2 对比度**：调一次 CSS token(--text-secondary 等)全站收敛 — 改动小收益大
3. **P0-3 色彩语义**：红色仅报警、降刺眼色 — 修 CHART token
4. **P1 焦点层次/字体**：逐页焦点卡瘦身 + 统一无衬线标题
5. 各页空状态(factory-3d/图表空白)补占位引导
